Project Description

This project targets to provide a free online platform for people to share their travel experience and memories, such as travel locations, stories, weather, and expenses, with the other users in the format of journals, pictures, or videos. Users would be able to post by locations, and they could also see other users' posts under the same location. This platform provides a great opportunity for people to better prepare for their future trips and to document their precious moments during a travel.

Project Design

  1. Technology
    • Netbeans IDE
    • Spring MVC
    • Maven
    • HTML
    • Database(MySQL)
    • External API
    • I will build up my project in Netbeans IDE using Maven and HTML, and I will use Spring MVC as the main framework as required by the project. We’ve used these technologies together in our past labs, and I believe they could be integrated. Building a database is something new to me, and as for now, I decide to use MySQL since it was something mentioned in class. For the external API, I'm thinking about using an external weather report website so the users could also see the weather at the place they are preparing to go to.
  2. Use case
    • Create profile
    • Login
    • Edit profile
    • Post journal/picture/video
    • Check weather at a location
    • Comment on posts
    • View posts by locations
    • View comments by posts
    • Delete/edit posts
    • Delete/edit comments
    • Logout
  3. Design details:
    • On the Home page, users would be able to see a map (not Google Map) with lots of famous travel locations, and they could select the location that they want to enter into to take a closer look. These locations that the users are currently seeing are just some places to start off designing our online platform with, but they will expand as more users are involved and more discussions become active. On the top right-hand corner, users would be able to login or register a new account.
    • The login and register buttons on the home page would each lead users to different pages:
      • Login page: the user would be able to enter username and password
      • Register page: the user could create a new account by entering username, password, first name, last name, email address and major/profession
    • After users click on a specific location, the system will lead them to the Discussion page for that topic. In the discussion page, users would be able to view all posts in chronological order. Besides this, they would be able to share new posts, and make comments to different posts.
    • In the top right-hand corner beside login/register buttons, users will see another button called “My Account”. Clicking on this button will lead them directly to the Account page. In the Account page, users would be able to perform activities listed below:
      • View/edit current information
      • View/edit/delete all posts
      • View/edit/delete all posted comments
    • On each page listed above, users would be able to see a tool bar on top left-hand corner on the page. The tool bar could lead users to Home page.
    • Above are the design details as now, and they might be changed and removed as the project goes. Meanwhile, additional functions may be added if needed.ol>
